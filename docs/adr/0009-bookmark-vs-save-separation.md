# ADR-0009：標記（折角）與儲存（收藏）分屬兩階段，不同時出現

- **狀態**：已採用（Adopted）
- **日期**：2026-07-20
- **決策者**：專案維護者
- **相關**：六個作答表面（歷屆原卷、模擬考、完整診斷、單題練習、錯題重練、弱點殲滅）
  與詳解檢視。

## 背景（Context）

兩個看似相近的動作其實服務不同階段：

- **標記**：作答途中對某題做的暫時記號（視覺為橘色折角），用途是長篇作答時
  「先跳過、待會回來檢查」——模擬紙本考卷折角的行為。
- **儲存**：對一題的長期收藏（如錯題本、重點題），是看完詳解後的整理動作。

若兩個按鈕在作答畫面同時出現，使用者難以分辨「這是待檢查記號還是收藏」，
且單題練習根本沒有「回頭檢查」的需求。

## 決策（Decision）

1. **標記只出現在長篇作答表面**（歷屆原卷、模擬考、完整診斷等多題連續作答）：
   作答中可加／取消橘折角，供導覽列快速跳回；**單題練習不顯示標記**。
2. **儲存只出現在詳解檢視**：作答送出、看詳解時才提供儲存按鈕。
3. **同一畫面永不同時出現兩者**：作答中只有標記，詳解中只有儲存。
4. 標記是**該次作答的狀態**：作答中存記憶體；交卷時隨該次歷史紀錄一併保存。
   之後在**歷史紀錄詳情**回看原始作答時，保留當時的折角，並可在該畫面新增／
   刪除標記（改動寫回該筆歷史紀錄）。標記永遠繫於「某一次作答」，不是跨卷的
   全域旗標；儲存才是跨卷長期收藏。

## 後果（Consequences）

- 作答 UI 保持單一動作，認知負擔低；「檢查用」與「收藏用」語意不混。
- 交卷後若使用者想長期收藏折角題，動線是：在成績／詳解頁逐題按「儲存」。
- 歷史詳情同時顯示「當次標記（可編輯）」與詳解區的「儲存」，兩者語意不同
  （前者屬於那一次作答，後者屬於題目本身），不視為違反「不同畫面」原則——
  該原則約束的是**作答中**的畫面。
- 標記持久化 schema＝歷史紀錄內一個 qid 陣列欄位（如 `flags`）。
