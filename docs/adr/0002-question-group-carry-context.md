# ADR-0002：題組／承上題的情境引入規則

- **狀態**：已採用（Adopted）——現行規則，非草案；細節取捨與待補工程見文末「未來工作」。
- **日期**：2026-07-16
- **決策者**：專案維護者
- **相關**：ADR-0001（計時的時間模型，另一份需要跨「作答表面」一致規則的決策）；
  程式位置 `web/run.js`（`groupHeaderEl`／`passageEl`／`carryContextEl`／`startSheet`／`startDrill`）、
  `web/modes.js`（組卷函式）；資料欄位 `data/<考試>/bank.json` 的 `group_id`／`passage`。

## 背景（Context）

題庫裡有兩種「不可單獨理解」的題：

- **資料層題組**：`bank.json` 有 `group_id`（編碼首末題號，如 `g23_25`、
  `會考_111_國文_g28_29`）與 `passage`（共用引文／資料本體）。
- **承上題**：題幹以「承上題」等詞開頭，情境在前一題題幹裡，資料層沒有連結欄位。

本站六個作答表面——歷屆原卷、模擬考、完整診斷、單題練習、錯題重練、弱點殲滅——與
歷史詳情，都可能以非原卷順序遇到這兩種題（混卷抽題、跨模式重練、單題拆出練習）。
「情境要不要補、補多少、補在哪」因此需要一份跨表面一致的規則，而不是每個表面各自
土法煉鋼。

## 決策（Decision）

**決策一：原文完整引入，禁止抽樣與摘要。**

情境框顯示錨題「題幹全文」。理由三點：

1. 連環題的作答關鍵常是錨題的單一數字或條件，節錄等於賭哪句不重要；
2. 摘要引入 AI 改寫層，違反「題庫忠於官方原文」的底線；
3. 依《著作權法》第 9 條，依法令舉行之考試試題不得為著作權標的，全文引用零法律成本。

唯一允許的處理是 `formatPassage` 的折行重組（純版面折行重組，不動內容）。

**決策二：只引題幹，不引選項與正解。**

選項與答案不是情境載體；引入會膨脹版面，且可能剝奪使用者之後獨立練到錨題的機會。

**決策三：鏈式堆疊。**

連環承上（例：33→34→35）顯示「錨題＋沿途每個中間承上題」的題幹，依原卷題號升冪、
各標「第 N 題」。

**決策四：模式分工。**

| 表面 | 情境處理 |
|---|---|
| 歷屆原卷 | 自然順序、零情境框（相鄰抑制：畫面上一題即錨題時不重複顯示） |
| 模擬考／完整診斷 | 抽題後做「鏈相鄰正規化」＋「錨題拉入」（見下） |
| 單題練習／錯題重練／弱點殲滅／歷史詳情 | 情境框自足、不聚攏（逐題節奏優先，每題須可獨立作答） |

- **鏈相鄰正規化**：同鏈／同 `group_id` 被抽中的成員聚攏到首個被抽中位置、依原題號
  升冪，於交錯／集中排序之後執行。
- **錨題拉入**：承上題被抽中而直接錨題落選時，用同科一題非題組題換入錨題，總題數與
  各科配比不變；換不進則退回情境框。

**決策五：觸發與解析判準。**

資料層（`group_id`／`passage`）優先於 runtime 回溯；兩者都沒有，且題幹命中 `CARRY_RE`
詞表（`web/run.js`）才走承上回溯：

> 承上題、依前文、依上文、承前題（「題」字可省，即「承前」亦符合）、同上題、依前題、
> 根據上題、接上題

回溯規則：在 `papersIndex` 的同一卷內往前走，到第一個非承上題為錨；找不到卷或找不到
錨題時，顯示誠實 fallback 文案，引導使用者去「歷屆原卷」看整卷。

詞表是正典：**新增觸發詞必須同步修訂本 ADR。**

## 考慮過的替代方案（Alternatives）

- **A. 摘要／抽樣式情境引入**：省版面，但賭哪句不重要，且摘要引入 AI 改寫層，違反
  「題庫忠於官方原文」的底線（決策一已排除）。
- **B. 情境框連選項／正解一起顯示**：省一次點擊，但膨脹版面、且可能剝奪使用者之後
  獨立練到錨題的機會（決策二已排除）。
- **C. 只顯示緊鄰前一題、不鏈式往上找**：多數情況夠用，但連環承上（例：33→34→35）
  會漏掉中間題的情境（決策三已排除）。
- **D. 所有表面統一處理**（例如歷屆原卷也做聚攏、或模擬考／完整診斷也不聚攏）：實作
  最簡單，但歷屆原卷聚攏會打亂真實卷序、模擬考／完整診斷不聚攏會讓使用者頻繁抽到
  承上題卻看不到情境（決策四的模式分工已排除）。

## 後果（Consequences）與驗證方式

- 情境框保證完整、不失真，但練習畫面偶爾會變長（尤其鏈式堆疊時）。
- 模擬考／完整診斷的「錨題拉入」會微調抽題結果（換入原本沒抽到的錨題），這是有意
  為之的取捨：情境完整優先於抽題結果的絕對隨機性。
- `CARRY_RE` 詞表若漏了官方考卷實際用語，該題會被誤判為一般題、不觸發回溯——這是
  本規則已知的邊界，見「未來工作」。

**可復刻的驗證配方**（正典測試題：`data/social-worker/bank.json` 的 `qid`
`111_111030_社會工作研究方法_33` 與 `111_111030_社會工作研究方法_34`——33 為錨題、
34 為承上題，34 的 stem 以「承上題，」開頭）：

1. **歷屆原卷**：該卷第 33、34 題相鄰出現，且皆無情境框。
2. **模擬考**：抽到 34 時，看得到 33 的題幹全文（情境框，或 33 本身被拉入同卷抽中
   位置）。
3. **單題練習**：遇到 34 時，情境框顯示 33 的題幹全文。

## 未來工作（Future Work）

長期應把承上鏈在 build 階段寫進資料層（如新增 `q.carry_of` 欄位，直接記錄錨題
`qid`），把「決策五」的 runtime 詞表回溯降級為資料層缺連結時的備援，而不是唯一機制
——詞表匹配終究是啟發式，抓不到官方用語的所有變體。遷移發生時，本 ADR 的「決策五」
判準節須隨之修訂。
