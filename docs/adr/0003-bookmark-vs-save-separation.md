# ADR-0003：標記（折角）與儲存（收藏）分屬兩階段，不同時出現

- **狀態**：已採用（Adopted）
- **日期**：2026-07-20
- **決策者**：專案維護者
- **相關**：六個作答表面（歷屆原卷、模擬考、完整診斷、單題練習、錯題重練、弱點殲滅）
  與詳解檢視。

## 背景（Context）

兩個看似相近的動作其實服務不同階段：

- **標記**：作答途中對某題做的暫時記號（視覺為橘色折角），用途是長篇作答時
  「先跳過、待會回來檢查」——模擬紙本考卷折角的行為。
- **儲存**：對一題的長期收藏（如錯題本、重點題），是看完詳解後的整理動作。

若兩個按鈕在作答畫面同時出現，使用者難以分辨「這是待檢查記號還是收藏」，
且單題練習根本沒有「回頭檢查」的需求。

## 決策（Decision）

1. **標記只出現在長篇作答表面**（歷屆原卷、模擬考、完整診斷等多題連續作答）：
   作答中可加／取消橘折角，供導覽列快速跳回；**單題練習不顯示標記**。
2. **儲存只出現在詳解檢視**：作答送出、看詳解時才提供儲存按鈕。
3. **同一畫面永不同時出現兩者**：作答中只有標記，詳解中只有儲存。
4. 標記是**卷內暫時狀態**——該次作答結束（交卷）後不必長期保留；
   儲存才進長期資料（localStorage／日後同步）。

## 後果（Consequences）

- 作答 UI 保持單一動作，認知負擔低；「檢查用」與「收藏用」語意不混。
- 交卷後若使用者想留住折角題，動線是：在成績／詳解頁逐題按「儲存」。
- 實作時標記狀態存於當次 session（記憶體即可），不需持久化 schema。
